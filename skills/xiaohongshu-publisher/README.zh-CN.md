# 小红书全自动发布 Skill（xiaohongshu-publisher）

这是一个面向“小红书内容从草稿到发布”的自动化技能，目标是把流程跑通、返工降到最低。

## 这个 Skill 做什么

- 接收任务并判定目标类型：草稿 / 改版 / 发布 / 资料页更新
- 自动整理并优化：标题、正文、图片顺序
- 先给用户预览完整草稿（标题+正文+图片）
- 用户确认后，进入 Creator Center 执行发布
- 发布后到管理页核验状态（审核中/已发布）并闭环汇报

## 核心流程（6步）

1. 明确目标（草稿/改版/发布/资料页）
2. 准备内容（标题、正文、配图顺序）
3. 发草稿给用户审
4. 确认后去发布页操作：
   - `https://creator.xiaohongshu.com/publish/publish`
5. 到管理页核验状态
6. 输出闭环：改动点 + 当前状态 + 下一步复查

## 自动化硬规则

- 任务开始必须报 ETA
- 多步骤过程中要报进度（最长静默不超过 3 分钟）
- **发布前必须拿到用户明确确认**（如“确认发布/发吧/可以发”）
- 收尾必须给：结果 + 状态 + 下一步

## 内容质量规则

- 标题建议 <= 20 个汉字
- 正文短句化，分段留白，避免“文字墙”
- 图片顺序发布前二次检查（封面→内容）
- 标签 3–6 个高相关即可

## 模板能力

内置模板文件：`references/templates.md`

包含：
- 标题模板
- 正文模板
- 置顶评论模板
- 发布闭环汇报模板

使用示例见：`USAGE.zh-CN.md`

## 适用场景

- “帮我写并发布一条小红书”
- “把这条小红书重写成可发版”
- “按模板批量改稿并逐条发布”
- “优化主页资料（头像/简介）”
