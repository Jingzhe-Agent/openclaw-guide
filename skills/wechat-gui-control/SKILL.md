# WeChat GUI Control

按下面流程执行，避免“看起来有@，实际没提及”的假成功。

## 硬规则（必须）

1. **外发前确认**：任何对外消息，先给用户看最终文案，拿到明确“发”再发送。
2. **真实@规则**：必须通过 `@` 候选列表点选成员，不接受手打 `@昵称`。
3. **双重校验**：
   - 点选后校验草稿中已出现目标成员名；
   - 发送前再校验一次两位成员都在提及位。
4. **单次发送**：未获新确认，不重复发送。
5. **默认预演模式**：除非用户在当前轮明确回复“发”，否则只执行到草稿与@校验，禁止回车发送。

## 标准流程

1. 先报预计耗时（一般 1-2 分钟）。
2. 聚焦微信窗口，进入目标群。
3. 点击输入框并清空草稿。
4. `@` + 点选第1个成员；校验已录入。
5. `@` + 点选第2个成员；校验已录入。
6. 粘贴正文（不覆盖已录入的@成员）。
7. 发前把最终文案回给用户确认（默认只预演，不发送）。
8. 未收到“发”：结束在预演态并回报“待确认”。
9. 收到“发”后再回车发送。
10. 发送后回报结果（done/blocked），若失败说明卡点。

## 实操要点

- 需要 OCR 时，用系统 Vision OCR（Swift + Vision）识别截图文本。
- 点击坐标要用**点坐标（point）**，不要直接用像素坐标（pixel）。
- 在 Retina 下常见换算：
  - `x_point = (x_norm + w_norm/2) * 1920`
  - `y_point = (1 - (y_norm + h_norm/2)) * 1080`
- 如果候选列表没弹出：先确保输入框有焦点，再输入 `@`。
- 如果识别不到全名：先输入关键词（如“迷路”“盖伦”）缩小候选后再点选。

## 完成回报模板

- 预计耗时：X 分钟
- 进度：已完成 A/B/C
- 结果：done 或 blocked
- 若 done：说明是否确认两位成员真实@成功
- 若 blocked：说明失败步骤 + 下一步建议
